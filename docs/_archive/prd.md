## 1. 产品定位

### 1.1 核心价值

**Neural Link 是一个全新的 AI 原生社交应用**，兼具社交和远程工作能力：

| 维度 | 描述 |
|------|------|
| **社交** | 类似 Discord/WhatsApp 的独立即时通讯平台 |
| **远程控制** | 手机远程控制自己的桌面电脑执行任务 |
| **AI 原生** | 智能回复建议、草稿确认、预测性操作 |

### 1.2 目标用户

- **主要用户**：需要远程控制自己电脑的职场人士
- **使用场景**：不在电脑前时，需要电脑执行工作（文件操作、Shell 命令、浏览器任务等）

## 3. 核心交互模式

### 3.1 Draft & Verify (草稿确认) [P0]

```
用户输入意图 → AI 生成草稿 → 用户确认 → 执行
```

**关键原则**：AI 永不自主行动，必须经过用户确认。

**示例流程**：
1. 用户在手机输入："重启后端服务"
2. 云端 AI 生成草稿：`ssh server "cd /app && pm2 restart backend"`
3. 手机端显示草稿卡片，用户点击"确认"
4. 桌面端执行命令，返回结果

### 3.2 The Whisper (耳语建议) [P1]

收到消息时，AI 生成 3 个简短回复建议（≤2 个气泡）：

```
收到消息 → 云端 AI 分析 → 推送 3 个建议 → 用户点击发送
```

**性能要求**：
- 延迟 < 800ms：显示建议
- 延迟 > 1000ms：放弃显示

### 3.3 Predictive Actions (预测执行) [P0]

AI 分析上下文，主动建议操作：

```
Shell 报错 → AI 生成修复卡片 → 用户点击 → 执行修复
```

**安全要求**：危险命令必须标记或阻止。

## 4. 首个里程碑 (MVP)

> **"手机 App 发送一个干活的指令给电脑端，电脑直接干活并且将任务交付，发回给手机端回复已经做完任务"**

**验收标准**：
1. 手机端发送工作指令
2. 桌面端接收并执行任务
3. 桌面端返回执行结果
4. 手机端显示任务完成状态

## 5. 非功能需求

|------|--------|

| 远程操作延迟 | < 3 秒 |
| AI 建议生成 | < 800ms |

### 5.2 平台支持

| 平台 | 优先级 |
|------|--------|
| iOS | P0 |
| Android | P0 |
| macOS | P0 |
| Windows | P0 |

### 5.3 安全要求

- 所有通信通过 WSS 加密
- 远程命令执行需要用户确认
- 危险操作需要二次确认
